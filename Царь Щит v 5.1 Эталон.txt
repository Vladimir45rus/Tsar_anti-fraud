**Версия:** 5.1 «АБСОЛЮТ»  
**Статус:** Единственный источник истины (Single Source of Truth). Финальная версия.  
**Назначение:** Полное техническое, бизнесовое и юридическое ТЗ для разработки, аудита, развёртывания, масштабирования и передачи сторонним исполнителям.  
**Приоритет:** Автономность, защита пользователя, неприкосновенность кода, финансовая устойчивость, юридическая чистота, человеческий фактор.

## I. ГЛОБАЛЬНАЯ ФИЛОСОФИЯ И НЕПОКОЛЕБИМЫЕ ЗАКОНЫ

1. **Цифровое противоядие:** Система не является пассивным определителем номера. Она работает на уровне смыслов, психологических паттернов и физиологических реакций, вырывая пользователя из гипноза социальной инженерии в реальном времени.
2. **Закон «Царской Крови» (Total Offline):** Ни один байт сырого аудиопотока не покидает устройство. Обработка происходит исключительно на процессоре/NPU телефона. На сервер передаются только хеши номеров, текстовые логи STT, метаданные срабатываний и статус лицензии.
3. **Закон 666 (Монолитность кода):** При любых обновлениях существующий код не урезается, не чистится и не модифицируется автоматически. Только добавление функций. Любые изменения вносятся строго с прямого разрешения владельца.
4. **Принцип Невидимости:** В фоне система потребляет 0% ресурсов 99% времени. ИИ-движок активируется только при событии входящего вызова с номером, отсутствующим в контактах, или при ручном запуске «Царского Теста».
5. **Принцип «Царского Слова» (Immutable UX):** Пользовательский интерфейс и ключевые логические цепочки не меняются без ведома владельца. Если ИИ обучается, то только в рамках весовых коэффициентов триггеров, не меняя архитектуру принятия решений.

## IV. ЛИЦЕНЗИРОВАНИЕ, ОПЛАТА И МЕХАНИКА АКТИВАЦИИ

### 4.1. Принцип «Один Номер = Одно Устройство»
Лицензия привязывается к связке `Номер телефона + Аппаратный отпечаток`. APK без активации работает в режиме «Тыква» (ограниченный оффлайн-функционал).

### 4.2. Пошаговый поток активации
1. Пользователь скачивает APK (RuStore / Сайт / Google Play Lite).
2. Первый запуск: экран «Царское соглашение» (явное согласие на обработку метаданных, доступ к контактам, оверлеи, алерты).
3. Генерация хеша: `SHA-256(Android_ID + Build.SERIAL + Brand + Model + CPU_ABI + Соль_сервера)`.
4. Запрос на сервер: `POST /api/v1/check_license { phone, device_hash, timestamp, signature }`.
5. Ответ сервера:
   - `STATUS: PAID` + совпадение хеша → выдаётся JWT-токен с `expiry: 72 часа`. Полная активация.
   - `STATUS: FREE` → режим «Демо» (оффлайн STT, белый список, 1 тестовый показ оверлея).
   - `STATUS: DEVICE_MISMATCH` → предложение переноса.

### 4.3. Перенос лицензии
- SMS-код подтверждения → деактивация старого хеша → активация нового → логирование → алерт семье. Лимит: 3 плановых переноса/год.
- При исчерпании всех плановых и экстренных лимитов — запрос в службу поддержки с ручной верификацией по видеосвязи (показ паспорта и лица). Только так восстанавливается доступ вне лимитов.

### 4.4. Экстренный протокол «Воскрешение»
При потере/краже/поломке устройства:
- Запрос SMS + ответ на секретный вопрос (задаётся при активации).
- Принудительная деактивация старого хеша, активация нового.
- Задержка 15 минут на отмену (окно безопасности).
- Лимит: 1 экстренный перенос/12 месяцев. При злоупотреблении → ручная верификация.

### 4.5. Таймер бездействия и Grace-Period
- >90 дней без связи с сервером → статус `SUSPICIOUS`. При следующем запуске требуется SMS-подтверждение.
- При внезапной потере сети работает «Мягкий период» (Grace-Period) 72 часа: защита функционирует на 100% на основе последнего валидного токена. Синхронизация происходит автоматически при появлении сети.

### 4.6. Резервный код восстановления
Генерируется при первой активации: `TSAR-XXXX-XXXX`. Сохраняется пользователем. Используется для мгновенного восстановления доступа без ожидания поддержки.

### 4.7. Триал-период 24 часа
После установки пользователь получает полный функционал на 24 часа. По истечении периода требуется оплата для продолжения использования премиум-функций.

### 4.8. Протокол «Наследие»
Доверенное лицо может запросить перенос или закрытие лицензии, предоставив свидетельство о смерти или нотариальную доверенность. После ручной проверки поддержка деактивирует аккаунт и выдаёт семейному кругу промокод на бесплатный год обслуживания.

## V. ЯДРО РАСПОЗНАВАНИЯ: OFFLINE-FIRST STT И МНОГОУРОВНЕВЫЙ ДЕТЕКТОР

### 5.1. Локальный STT и Грациозная Деградация
- Модель квантована под NPU. Задержка 80-150 мс. Аудио буферизируется в RAM, передаётся модели, текст удаляется после анализа.
- **Поддержка слабых устройств:** При RAM < 3 ГБ автоматически загружается ultra-lite модель (меньше словарь, выше скорость).
- **Каскадная деградация:** Если основная модель (Vosk) не загружается (ошибка совместимости NPU), система переключается на Silero. Если и Silero не грузится → включается режим «Акустический страж» (только поведенческий и стресс-детектор без лексического анализа, но с боевыми режимами). Пользователь получает уведомление: «Модуль распознавания речи не загружен. Защита работает в упрощённом режиме».

### 5.2. Многоуровневый детектор угрозы
1. **Лексический:** 500+ паттернов и контекстных связок. Обновляется раз в сутки. В оффлайн-режиме используется последний кэш. Паттерны: «Госуслуги» + «взлом», «МВД» + «уголовное дело», «Банк» + «безопасный счет», «СМС» + «код», «Внук» + «ДТП».
2. **Поведенческий:** Анализ темпа речи (монолог-давление >40 сек без пауз), частота перебиваний, асинхронность диалога.
3. **Стресс-Детектор:** Анализ микро-колебаний голоса владельца (дрожание, падение громкости, прерывистость). Комбинация pitch-tracking + анализ пауз. Порог настраивается пользователем.
4. **Технический (IVR-детектор):** Анализ спектра на предмет идеально ровного цифрового голоса без микропауз, характерного для синтеза речи/роботов. Если обнаружен робот + стоп-фразы → включение режима «Ложный Обрыв» немедленно, без эскалации.

### 5.3. Персональные стоп-фразы
- В приложении «Приёмник» пользователь может добавить 5-10 персональных стоп-слов (кличка питомца, девичья фамилия, название улицы). При их обнаружении — мгновенная эскалация до «Красного Кода» без дополнительного анализа.

### 5.4. Динамический «Царский Тест»
- Активируется ТОЛЬКО если контакт переименован в `«ТЕСТ ЗВОНОК»`.
- Фраза-пароль запускает демо (красный экран, вибро, голос).
- Если мошенник произнесёт «тест звонок» с левого номера → срабатывает боевой режим. Словарь не раскрывается.

## VII. СИСТЕМА «СЕМЬЯ» И ПРИЛОЖЕНИЕ-ПРИЁМНИК

### 7.1. Приложение «Царь-Щит Приёмник»
- Нативное приложение-компаньон для доверенных лиц. Работает через Push вашего сервера + SMS + Email-дублирование + голосовой робот.
- Голосовые уведомления реализуются через сертифицированные шлюзы (Т-Банк, Яндекс, специализированные SMS-шлюзы) для гарантии доставки и соответствия ФЗ «О рекламе»/правилам операторов.

### 7.2. Функционал Приёмника
- Текстовый STT-лог в реальном времени (без аудиофайлов).
- Кнопка «Принудительно завершить вызов» → отправка команды → эмуляция нажатия «Положить трубку» через Accessibility. Требует явной галочки «Разрешить доверенному лицу завершать вызовы» в основном приложении. В момент нажатия на целевом устройстве появляется плашка: «Вызов завершён доверенным лицом. Ожидайте звонка».
- Гео-позиция (только при явном разрешении в основном приложении).
- Экспорт PDF-отчёта для полиции.
- Управление лицензиями в семейном пакете.
- Добавление персональных стоп-фраз.

### 7.3. Резервный мульти-канал доставки
1. Push от сервера.
2. Если не доставлено за 30 сек → SMS с кодом-триггером.
3. Если не прочитано → Email с темой `[СРОЧНО] Царь-Щит: Атака`.
4. Если всё молчит 60 сек → исходящий вызов с голосовым роботом: «У вашего отца зафиксирована атака. Подтвердите получение нажатием 1.»

### 7.4. Сертификат безопасности
После успешной установки и прохождения «Царского Теста» генерируется изображение: «Абонент [Имя] находится под защитой Царь-Щита. Уровень защиты: Максимальный». За расшаривание начисляются «Царь-Баллы» для вирусного роста.

```
       ↓
[Пользователи] → [Не замечают простоя (<2 мин)]
```

### 9.4. Режим «Тихий мониторинг» сервера
При нагрузке CPU >80% автоматически отключаются не-критичные задачи (обновление базы репутации, аналитика). Все ресурсы направляются на проверку лицензий и обработку алертов.

### 9.5. Экстренный план коммуникации
При падении сервера пользователи видят статичную страницу-заглушку: «Царь-Щит на техническом обслуживании. Ваша защита активна. Ожидайте». Это сохраняет доверие и снижает нагрузку на поддержку.

## X. ЗАЩИТА КОДА И ИНЖЕНЕРНАЯ КРЕПОСТЬ

1. **Закрытый исходный код:** Приватный репозиторий, доступ по 2FA. Никаких Open Source ядер в боевых модулях.
2. **Обфускация «Лабиринт»:** R8/ProGuard + кастомный шифратор строк + Control Flow Obfuscation. Динамическая обфускация эндпоинтов: `базовый_url + SHA256(дата + секретное_слово)`. Все ключевые фразы и API-эндпоинты шифруются на этапе сборки, расшифровываются в runtime.
3. **Hardware Keystore:** Ключ `Tsarskiy_Key_2026` хранится в Android Keystore / Secure Enclave. Невозможно извлечь даже с Root.
4. **Режим «Сон-Трава» (Honeypot) + Runtime Integrity Check:** При несоответствии CRC/SHA-256 подписи APK или обнаружении запущенных отладчиков/перехватчиков трафика (Frida, Charles, HTTP Toolkit) приложение переходит в симуляцию. Каждые 15 минут отправляет на сервер-ловушку: IP, IMEI, Geo, модель, список инструментов взлома. Устройство получает вечный бан.
5. **Защита от удаления («Анти-Стирание»):** Запрос прав Device Admin + перехват события удаления через Accessibility. Требует PIN или SMS-подтверждение. При попытке → алерт семье + логирование.

## XI. ЮРИДИЧЕСКАЯ БРОНЯ И ПОЛИТИКА ДАННЫХ

- **Царское соглашение:** При первом запуске явное подтверждение на обработку аудиопотока (локально), доступ к контактам, оверлеи, отправку алертов. Защита от 152-ФЗ и ст. 138 УК РФ.
- **Локальный криптоконтейнер:** Логи шифруются AES-256, ключ привязан к биометрии/пин-коду. Автоматическая ротация/очистка через 7 дней. Сырое аудио никогда не сохраняется.
- **Нулевая передача без действия:** На сервер уходят только хеши номеров, метаданные срабатываний, текстовые логи STT, статус лицензии. Никаких персональных данных без явного разрешения.

## ПРИЛОЖЕНИЕ В: СХЕМА БАЗЫ ДАННЫХ (PostgreSQL — УПРОЩЁННО)

```sql
-- Пользователи и лицензии
CREATE TABLE users (
    id BIGSERIAL PRIMARY KEY,
    phone_hash VARCHAR(64) UNIQUE NOT NULL,
    created_at TIMESTAMP DEFAULT NOW(),
    plan_type VARCHAR(20) DEFAULT 'personal',
    status VARCHAR(20) DEFAULT 'free'
);

CREATE TABLE licenses (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT REFERENCES users(id),
    device_hash VARCHAR(64) NOT NULL,
    activated_at TIMESTAMP DEFAULT NOW(),
    expires_at TIMESTAMP,
    last_sync TIMESTAMP,
    transfer_count INT DEFAULT 0,
    emergency_transfer_count INT DEFAULT 0,
    UNIQUE(user_id, device_hash)
);

CREATE TABLE triggers (
    id BIGSERIAL PRIMARY KEY,
    category VARCHAR(50) NOT NULL,
    phrase TEXT NOT NULL,
    weight FLOAT DEFAULT 1.0,
    is_active BOOLEAN DEFAULT true,
    updated_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE reputation (
    id BIGSERIAL PRIMARY KEY,
    phone_hash VARCHAR(64) UNIQUE NOT NULL,
    report_count INT DEFAULT 0,
    last_reported TIMESTAMP,
    is_toxic BOOLEAN DEFAULT false
);

CREATE TABLE incidents (
    id BIGSERIAL PRIMARY KEY,
    user_hash VARCHAR(64) NOT NULL,
    phone_hash VARCHAR(64) NOT NULL,
    trigger_category VARCHAR(50),
    stt_log_hash VARCHAR(64),
    timestamp TIMESTAMP DEFAULT NOW(),
    device_model VARCHAR(100)
);

CREATE TABLE loyalty (
    user_id BIGINT REFERENCES users(id),
    points INT DEFAULT 0,
    level VARCHAR(20) DEFAULT 'newbie',
    last_updated TIMESTAMP DEFAULT NOW()
);
```
